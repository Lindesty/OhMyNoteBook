## 方法不能自动 `overload`

在typescript中方法只支持多个签名 + 一个实现，这意味着你只能为类写一个构造方法

下面的代码是不合法的❌

```ts
function fn(a: number) {
  console.log(a);
}
function fn(a: string) {
  console.log(a);
}
```

需要改写成下面这样

```ts
function fn(a: number): number; // 重载签名 1
function fn(a: string): string; // 重载签名 2
function fn(a: number | string) {
  // 唯一实现
  console.log(a);
}
```

## 类型与接口并没有签名

如下面的代码，类型系统是结构化类型，不是名义类型，两相结构相同的类型会被视为兼容

```ts
interface A {
  x: number;
}
interface B {
  x: number;
}

let a: A = { x: 1 };
let b: B = a; // ✔ 合法（结构兼容）
```

## 枚举(Enum)

TS enum 是运行时对象，会生成 JS 代码，现在仍有许多打包工具不兼容

更糟的是，enum有反射映射

```ts
enum A {
  X,
}
A[0] === "X"; // true
```

建议使用客串字面量联合类型(Union Type)

```
// 定义类型
type UserRole = 'Admin' | 'Editor' | 'Viewer';

function setUserRole(role: UserRole) {
  // ...
}

// 使用
setUserRole('Admin'); // ✅ 类型安全
```

也可使用常量对象 + as const

```ts
// 定义常量对象和类型
const Status = {
  Pending: 'PENDING',
  Success: 'SUCCESS',
  Fail: 'FAIL',
} as const;

// 从对象值中推断出类型: "PENDING" | "SUCCESS" | "FAIL"
type Status = typeof Status[keyof typeof Status];

function handleStatus(status: Status) {
  // ...
}

// 使用
handleStatus(Status.Success); // ✅ 类型安全且可遍历
Object.values(Status); // ✅ 可以遍历
```

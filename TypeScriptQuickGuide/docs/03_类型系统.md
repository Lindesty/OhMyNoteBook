# 类型系统

TypeScript 的类型系统是其核心特性。与 Java/C# 的名义类型不同，TypeScript 使用结构化类型（鸭子类型），关注类型的结构而非名称。

## 原始类型

```ts
// 字符串
const name: string = "Tom";

// 数字（所有数字都是固定64位浮点数）
const age: number = 30;
const price: number = 99.99;
const hex: number = 0xff;

// 布尔值
const active: boolean = true;

// null 和 undefined
const empty: null = null;
const notDefined: undefined = undefined;

// BigInt（可变长大整数）
const big: bigint = 9007199254740991n;

// Symbol（唯一标识符）
const key: symbol = Symbol("key");
```

## 对象类型

### 接口（interface）

```ts
interface User {
  id: number;
  name: string;
  email?: string;           // 可选属性
  readonly createdAt: Date; // 只读属性
}

const user: User = {
  id: 1,
  name: "Tom",
  createdAt: new Date()
};
```

### 类型别名（type）

```ts
type ID = string | number;
type Point = { x: number; y: number };
type Callback = (data: string) => void;

// 接口和类型别名的区别
// interface: 可以被扩展和实现
// type: 更灵活，支持联合类型、映射类型等
```

### 类（class）

```ts
class Person {
  // 属性
  private _name: string;
  public age: number;

  // 构造函数
  constructor(name: string, age: number) {
    this._name = name;
    this.age = age;
  }

  // getter
  get name(): string {
    return this._name;
  }

  // 方法
  greet(): string {
    return `Hello, I'm ${this._name}`;
  }
}
```

## 数组类型

```ts
// 方式1：元素类型[]
const numbers: number[] = [1, 2, 3];
const names: string[] = ["Tom", "Jerry"];

// 方式2：泛型数组
const items: Array<number> = [1, 2, 3];

// 只读数组
const readonly: readonly number[] = [1, 2, 3];
```

## 元组类型

固定长度和类型的数组：

```ts
// 基本元组
const point: [number, number] = [10, 20];

// 带标签的元组
const labeled: [x: number, y: number] = [10, 20];

// 可选元素
const optional: [string, number?] = ["hello"];

// 剩余元素
const rest: [number, ...string[]] = [1, "a", "b", "c"];
```

## 联合类型

表示可以是多种类型之一：

```ts
type StringOrNumber = string | number;

function process(value: StringOrNumber) {
  if (typeof value === "string") {
    console.log(value.toUpperCase());
  } else {
    console.log(value.toFixed(2));
  }
}

// 字面量联合类型
type Status = "pending" | "approved" | "rejected";
type Direction = "up" | "down" | "left" | "right";
```

## 交叉类型

合并多个类型：

```ts
interface Name {
  name: string;
}

interface Age {
  age: number;
}

type Person = Name & Age;

const person: Person = {
  name: "Tom",
  age: 30
};
```

## 字面量类型

精确的值作为类型：

```ts
// 字符串字面量
type Greeting = "hello" | "hi" | "hey";

// 数字字面量
type Dice = 1 | 2 | 3 | 4 | 5 | 6;

// 布尔字面量
type YesNo = true | false;

// 对象字面量类型
type Point = { x: 0; y: 0 };
const origin: Point = { x: 0, y: 0 };
```

## any、unknown、never

### any

关闭类型检查（不推荐）：

```ts
let anything: any = "hello";
anything = 123;
anything.toUpperCase(); // 编译通过，但运行时可能报错
```

### unknown

类型安全的 any：

```ts
let value: unknown = "hello";

// 不能直接使用
// value.toUpperCase(); // ❌ 错误

// 必须先检查类型
if (typeof value === "string") {
  console.log(value.toUpperCase()); // ✅
}
```

### never

永不存在的类型：

```ts
// 永不返回的函数
function throwError(message: string): never {
  throw new Error(message);
}

// 不可能的联合类型
type Impossible = string & number; // never

// 穷尽性检查
type Shape = "circle" | "square";
function area(shape: Shape): number {
  switch (shape) {
    case "circle": return 1;
    case "square": return 1;
    default:
      const _exhaustive: never = shape; // 确保所有情况都处理
      throw new Error("Unknown shape");
  }
}
```

## 类型断言

告诉编译器你比它更了解类型：

```ts
// as 语法
const value: unknown = "hello";
const str = value as string;

// 尖括号语法（JSX 中不推荐）
const num = <number>value;

// 非空断言
let maybeNull: string | null = "hello";
const definitelyString: string = maybeNull!;

// const 断言
const config = {
  url: "https://api.example.com"
} as const;
// { readonly url: "https://api.example.com" }
```

## 可空类型

```ts
// 可选属性
interface User {
  name: string;
  email?: string; // string | undefined
}

// 联合 null
let name: string | null = null;

// 非空检查
function greet(name: string | null) {
  if (name !== null) {
    console.log(name.toUpperCase());
  }
}
```

## 函数类型

```ts
// 函数类型定义
type Callback = (data: string) => void;
type Handler = (event: Event) => boolean;

// 函数签名
interface SearchFunc {
  (source: string, subString: string): boolean;
}

// 构造签名
interface Constructor {
  new (name: string): User;
}
```

## 类型兼容性规则

TypeScript 使用结构化类型：

```ts
interface Point2D { x: number; y: number; }
interface Point3D { x: number; y: number; z: number; }

const point3D: Point3D = { x: 1, y: 2, z: 3 };
const point2D: Point2D = point3D; // ✅ Point3D 有所有 Point2D 的属性
```

## 与 Java/C# 类型系统对比

| 特性 | Java | C# | TypeScript |
|------|------|-----|------------|
| 类型判断 | 名义类型 | 名义类型 | 结构类型 |
| null/undefined | null 是独立类型 | null 是独立类型 | null/undefined 可选 |
| 可选类型 | Optional<T> | Nullable<T> | ? 修饰符 |
| 联合类型 | 不支持 | 不支持 | A \| B |
| 交叉类型 | 不支持 | 不支持 | A & B |
| 类型推断 | 有限支持 | var 关键字 | 全面支持 |
| 泛型 | 类型擦除 | 运行时保留 | 类型擦除 |

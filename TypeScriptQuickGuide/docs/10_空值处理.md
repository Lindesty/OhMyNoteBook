# 空值处理

TypeScript 提供了多种处理 null 和 undefined 的方式。与 Java/C# 的 Optional/Nullable 不同，TypeScript 使用联合类型和特殊的运算符。

## null 和 undefined

### 区别

```ts
// null: 明确的空值
let empty: null = null;

// undefined: 未定义的值
let notDefined: undefined = undefined;

// 未初始化的变量
let declared: string;
console.log(declared); // undefined

// 缺失的属性
const obj = { name: "Tom" };
console.log(obj.email); // undefined
```

### strictNullChecks

```ts
// strictNullChecks: false（不推荐）
let name: string = null; // ✅ 允许

// strictNullChecks: true（推荐）
let name: string = null; // ❌ 错误
let name: string | null = null; // ✅ 必须显式声明
```

## 可选属性

```ts
interface User {
  name: string;
  email?: string; // 等价于 email: string | undefined
}

const user: User = { name: "Tom" };
console.log(user.email); // string | undefined
```

## 可选链（?.）

```ts
interface User {
  name: string;
  address?: {
    city?: string;
  };
}

const user: User = { name: "Tom" };

// 安全访问嵌套属性
const city = user.address?.city; // string | undefined

// 安全调用方法
const result = user.address?.toString?.();

// 安全访问数组
const items: string[] | undefined = undefined;
const first = items?.[0]; // undefined
```

## 空值合并（??）

```ts
// 只对 null 和 undefined 生效
const a = null ?? "default";      // "default"
const b = undefined ?? "default"; // "default"
const c = "" ?? "default";        // ""（空字符串不触发）
const d = 0 ?? "default";         // 0（零不触发）

// 与 || 的区别
const e = 0 || "default";  // "default"（0 是 falsy）
const f = 0 ?? "default";  // 0
```

## 非空断言（!）

```ts
function process(name: string | null) {
  // 告诉编译器 name 不为 null
  console.log(name!.toUpperCase());
}

// 使用场景：确定值不为空时
const element = document.getElementById("app")!;
element.innerHTML = "Hello";
```

## 类型守卫

### 手动检查

```ts
function greet(name: string | null | undefined) {
  if (name === null || name === undefined) {
    return "Hello, Guest";
  }
  return `Hello, ${name.toUpperCase()}`;
}

// 更简洁的写法
function greet2(name: string | null | undefined) {
  if (name == null) { // 同时检查 null 和 undefined
    return "Hello, Guest";
  }
  return `Hello, ${name.toUpperCase()}`;
}
```

### typeof 检查

```ts
function process(value: string | null) {
  if (typeof value === "string") {
    console.log(value.toUpperCase()); // value: string
  }
}
```

### 真值检查

```ts
function process(value: string | null | undefined) {
  if (value) {
    console.log(value.toUpperCase()); // 排除了 null、undefined、""
  }
}
```

## 工具类型

### NonNullable

```ts
type T = string | number | null | undefined;
type NonNullableT = NonNullable<T>; // string | number
```

### 自定义 NonNullable 类型守卫

```ts
function isDefined<T>(value: T | null | undefined): value is T {
  return value !== null && value !== undefined;
}

const items: (string | null | undefined)[] = ["a", null, "b", undefined];
const defined = items.filter(isDefined); // string[]
```

## 与 Java/C# 的对比

| 特性 | Java | C# | TypeScript |
|------|------|-----|------------|
| 可空类型 | `Optional<T>` | `Nullable<T>` / `T?` | `T \| null` |
| 安全访问 | `Optional.map()` | `?.` | `?.` |
| 默认值 | `Optional.orElse()` | `??` | `??` |
| 非空断言 | `Optional.get()` | `!` | `!` |
| 检查是否为空 | `Optional.isEmpty()` | `HasValue` | `== null` |

**Java 示例**：
```java
Optional<String> name = Optional.ofNullable(getName());
String greeting = name
    .map(n -> "Hello, " + n)
    .orElse("Hello, Guest");
```

**TypeScript 等价**：
```ts
const name = getName();
const greeting = name != null
    ? `Hello, ${name}`
    : "Hello, Guest";

// 或使用可选链和空值合并
const greeting = `Hello, ${name ?? "Guest"}`;
```

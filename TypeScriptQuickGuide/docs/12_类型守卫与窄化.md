# 类型守卫与窄化

类型守卫（Type Guard）和类型窄化（Type Narrowing）是 TypeScript 特有的概念，用于在运行时检查值的类型，让编译器在特定代码块中推断出更精确的类型。对于来自 Java/C# 的开发者，这是 TypeScript 动态类型系统带来的独特特性。

## 为什么需要类型守卫

TypeScript 是静态类型系统，但编译后的 JavaScript 没有类型信息。当处理动态数据时，我们需要在运行时检查类型：

```ts
function process(value: string | number) {
  // value.toFixed() // ❌ 错误：string 上不存在 toFixed
  // value.toUpperCase() // ❌ 错误：number 上不存在 toUpperCase

  // 需要先判断类型
  if (typeof value === "string") {
    console.log(value.toUpperCase()); // ✅ value 被窄化为 string
  } else {
    console.log(value.toFixed(2)); // ✅ value 被窄化为 number
  }
}
```

## typeof 类型守卫

`typeof` 操作符是最基本的类型守卫，适用于原始类型。

```ts
function process(value: string | number | boolean) {
  if (typeof value === "string") {
    return value.toUpperCase(); // value: string
  } else if (typeof value === "number") {
    return value.toFixed(2); // value: number
  } else {
    return value; // value: boolean
  }
}
```

### typeof 的局限性

`typeof` 只能区分基本类型：

```ts
typeof "hello"       // "string"
typeof 123           // "number"
typeof true          // "boolean"
typeof undefined     // "undefined"
typeof Symbol()      // "symbol"
typeof 123n          // "bigint"
typeof {}            // "object"
typeof []            // "object" ❗ 数组也是 object
typeof null          // "object" ❗ null 也是 object
typeof function(){}  // "function"
```

对于对象、数组、null，`typeof` 无法区分，需要其他方法。

## instanceof 类型守卫

`instanceof` 用于检查对象是否是某个类的实例。

```ts
class Dog {
  bark() { console.log("Woof!"); }
}

class Cat {
  meow() { console.log("Meow!"); }
}

function makeSound(animal: Dog | Cat) {
  if (animal instanceof Dog) {
    animal.bark(); // animal: Dog
  } else {
    animal.meow(); // animal: Cat
  }
}
```

### 使用内置类

```ts
function process(value: Date | string | RegExp) {
  if (value instanceof Date) {
    return value.toISOString(); // value: Date
  } else if (value instanceof RegExp) {
    return value.test("pattern"); // value: RegExp
  } else {
    return value.toUpperCase(); // value: string
  }
}
```

### 局限性

`instanceof` 只能用于类，不能用于接口或类型别名（编译后不存在）：

```ts
interface Animal { name: string; }
class Dog implements Animal { name = "Dog"; }

const dog: Animal = new Dog();
// dog instanceof Animal // ❌ 错误：Animal 是接口，不是类
dog instanceof Dog // ✅
```

## in 操作符守卫

`in` 操作符检查属性是否存在于对象中，适用于区分不同结构的对象。

```ts
interface Bird {
  fly(): void;
  layEggs(): void;
}

interface Fish {
  swim(): void;
  layEggs(): void;
}

function move(animal: Bird | Fish) {
  if ("fly" in animal) {
    animal.fly(); // animal: Bird
  } else {
    animal.swim(); // animal: Fish
  }
}
```

### 可选属性

```ts
interface User {
  name: string;
  email?: string;
}

function getEmail(user: User): string {
  if ("email" in user) {
    return user.email; // user.email: string（不是 undefined）
  }
  return "no-email@example.com";
}
```

## 自定义类型谓词

当内置的类型守卫不够用时，可以定义自己的类型守卫函数。

### 基本语法

```ts
function isString(value: unknown): value is string {
  return typeof value === "string";
}

function process(value: unknown) {
  if (isString(value)) {
    console.log(value.toUpperCase()); // value: string
  }
}
```

`value is string` 就是类型谓词，告诉编译器：如果函数返回 true，value 就是 string 类型。

### 实际应用

```ts
interface Admin {
  role: "admin";
  permissions: string[];
}

interface User {
  role: "user";
  email: string;
}

function isAdmin(user: Admin | User): user is Admin {
  return user.role === "admin";
}

function showInfo(user: Admin | User) {
  if (isAdmin(user)) {
    console.log("Permissions:", user.permissions); // user: Admin
  } else {
    console.log("Email:", user.email); // user: User
  }
}
```

### 断言函数

使用 `asserts` 关键字，当条件不满足时抛出错误：

```ts
function assertIsString(value: unknown): asserts value is string {
  if (typeof value !== "string") {
    throw new Error("Value is not a string");
  }
}

function process(value: unknown) {
  assertIsString(value);
  // 如果执行到这里，value 一定是 string
  console.log(value.toUpperCase());
}
```

### 断言条件

```ts
function assertDefined<T>(value: T | null | undefined): asserts value is T {
  if (value === null || value === undefined) {
    throw new Error("Value is null or undefined");
  }
}

function getLength(str: string | null) {
  assertDefined(str);
  return str.length; // str: string
}
```

## 可辨识联合

可辨识联合（Discriminated Unions）是一种模式，使用公共属性来区分联合类型的不同变体。

### 基本模式

```ts
// 使用 kind/literal 字段作为辨识标志
interface Circle {
  kind: "circle";
  radius: number;
}

interface Square {
  kind: "square";
  sideLength: number;
}

interface Rectangle {
  kind: "rectangle";
  width: number;
  height: number;
}

type Shape = Circle | Square | Rectangle;

function getArea(shape: Shape): number {
  switch (shape.kind) {
    case "circle":
      return Math.PI * shape.radius ** 2; // shape: Circle
    case "square":
      return shape.sideLength ** 2; // shape: Square
    case "rectangle":
      return shape.width * shape.height; // shape: Rectangle
  }
}
```

### 实际应用：Redux Action

```ts
interface AddTodoAction {
  type: "ADD_TODO";
  payload: { text: string };
}

interface RemoveTodoAction {
  type: "REMOVE_TODO";
  payload: { id: number };
}

interface ToggleTodoAction {
  type: "TOGGLE_TODO";
  payload: { id: number };
}

type TodoAction = AddTodoAction | RemoveTodoAction | ToggleTodoAction;

function todoReducer(state: Todo[], action: TodoAction): Todo[] {
  switch (action.type) {
    case "ADD_TODO":
      return [...state, { id: Date.now(), text: action.payload.text }];
    case "REMOVE_TODO":
      return state.filter(todo => todo.id !== action.payload.id);
    case "TOGGLE_TODO":
      return state.map(todo =>
        todo.id === action.payload.id
          ? { ...todo, completed: !todo.completed }
          : todo
      );
  }
}
```

### 实际应用：API 响应

```ts
interface SuccessResponse<T> {
  status: "success";
  data: T;
}

interface ErrorResponse {
  status: "error";
  error: { code: number; message: string };
}

interface LoadingResponse {
  status: "loading";
}

type ApiResponse<T> = SuccessResponse<T> | ErrorResponse | LoadingResponse;

function handleResponse<T>(response: ApiResponse<T>) {
  switch (response.status) {
    case "success":
      console.log("Data:", response.data);
      break;
    case "error":
      console.error("Error:", response.error.message);
      break;
    case "loading":
      console.log("Loading...");
      break;
  }
}
```

## 类型断言

当开发者比编译器更了解类型时，可以使用类型断言。

### as 语法

```ts
const value: unknown = "hello";
const str = value as string; // 告诉编译器：我知道这是 string
console.log(str.toUpperCase());
```

### 尖括号语法

```ts
const value: unknown = "hello";
const str = <string>value; // 等价于 value as string
```

> 注意：在 TSX（React）文件中，尖括号语法会与 JSX 冲突，应使用 `as` 语法。

### 非空断言

```ts
function getName(name: string | null): string {
  return name!.toUpperCase(); // 告诉编译器：name 不会是 null
}

// 或者更安全的方式
function getNameSafe(name: string | null): string {
  return name?.toUpperCase() ?? "UNKNOWN";
}
```

### 常见问题

类型断言不会进行运行时检查：

```ts
const value: unknown = 123;
const str = value as string; // 编译通过
console.log(str.toUpperCase()); // 运行时报错！
```

**正确做法**：先检查再断言，或使用类型守卫。

## 穷尽性检查

使用 `never` 类型确保所有情况都被处理。

### 基本模式

```ts
type Shape = Circle | Square | Rectangle;

function getArea(shape: Shape): number {
  switch (shape.kind) {
    case "circle":
      return Math.PI * shape.radius ** 2;
    case "square":
      return shape.sideLength ** 2;
    case "rectangle":
      return shape.width * shape.height;
    default:
      // 如果有未处理的情况，这里会编译报错
      const _exhaustiveCheck: never = shape;
      throw new Error(`Unknown shape: ${_exhaustiveCheck}`);
  }
}
```

### 使用函数

```ts
function assertNever(value: never): never {
  throw new Error(`Unexpected value: ${value}`);
}

function getArea(shape: Shape): number {
  switch (shape.kind) {
    case "circle":
      return Math.PI * shape.radius ** 2;
    case "square":
      return shape.sideLength ** 2;
    // 假设忘记处理 rectangle
    default:
      return assertNever(shape); // ❌ 编译错误！
  }
}
```

如果后来添加了新的联合类型成员但忘记处理：

```ts
type Shape = Circle | Square | Rectangle | Triangle; // 新增 Triangle

// switch 中没有处理 triangle
// default 分支中 shape 的类型是 Triangle（不是 never）
// assertNever(shape) 会编译报错
```

## 类型窄化的方式汇总

### 1. typeof

```ts
if (typeof x === "string") { /* x: string */ }
```

### 2. instanceof

```ts
if (x instanceof Date) { /* x: Date */ }
```

### 3. in 操作符

```ts
if ("name" in obj) { /* obj 有 name 属性 */ }
```

### 4. 相等性检查

```ts
function process(x: string | number, y: string | boolean) {
  if (x === y) {
    // x 和 y 都是 string
    // 因为只有 string 是两个类型的交集
  }
}
```

### 5. 类型谓词

```ts
if (isString(x)) { /* x: string */ }
```

### 6. 可辨识联合

```ts
switch (shape.kind) {
  case "circle": /* shape: Circle */
}
```

### 7. 数组过滤

```ts
const items: (string | number)[] = [1, "a", 2, "b"];
const strings = items.filter((item): item is string => typeof item === "string");
// strings: string[]
```

### 8. 赋值窄化

```ts
let x: string | number;
x = "hello";
console.log(x.toUpperCase()); // x: string
```

### 9. truthy 检查

```ts
function process(str: string | null | undefined) {
  if (str) {
    console.log(str.toUpperCase()); // str: string（排除了 null/undefined/""）
  }
}
```

## 与 Java/C# 的对比

| 特性 | Java/C# | TypeScript |
|------|---------|------------|
| 类型检查 | 编译时 + 运行时（RTTI） | 仅编译时（类型擦除） |
| instanceof | 运行时检查类类型 | 运行时检查原型链 |
| 接口检查 | instanceof 不适用接口 | 使用 in 操作符检查属性 |
| 模式匹配 | switch + instanceof | 可辨识联合 + switch |
| 类型转换 | 显式转型，运行时检查 | 类型断言，无运行时检查 |
| 空值检查 | Optional / Nullable<T> | null/undefined + 可选链 |

## 最佳实践

### 1. 优先使用可辨识联合

```ts
// ❌ 使用多个类型守卫
interface User {
  isAdmin?: boolean;
  permissions?: string[];
  email?: string;
}

function hasPermission(user: User): boolean {
  if (user.isAdmin && user.permissions) {
    return user.permissions.includes("admin");
  }
  return false;
}

// ✅ 使用可辨识联合
interface Admin {
  role: "admin";
  permissions: string[];
}

interface Member {
  role: "member";
  email: string;
}

type User = Admin | Member;

function hasPermission(user: User): boolean {
  if (user.role === "admin") {
    return user.permissions.includes("admin");
  }
  return false;
}
```

### 2. 避免过度使用类型断言

```ts
// ❌ 不安全的断言
const value = JSON.parse(data) as User;

// ✅ 使用类型守卫验证
function isUser(value: unknown): value is User {
  return typeof value === "object"
    && value !== null
    && "name" in value
    && "email" in value;
}

const value = JSON.parse(data);
if (isUser(value)) {
  // 安全使用
}
```

### 3. 使用穷尽性检查

```ts
// 在 switch 的 default 分支中使用 never
// 确保所有情况都被处理
```

### 4. 结合可选链和空值合并

```ts
// 现代简洁的写法
const name = user?.profile?.name ?? "Unknown";

// 而不是
let name = "Unknown";
if (user && user.profile && user.profile.name) {
  name = user.profile.name;
}
```

# 泛型

泛型是 TypeScript 中实现代码复用的核心机制。如果你熟悉 Java 或 C# 的泛型，TypeScript 的泛型概念会非常容易理解，但在语法和一些行为上存在重要差异。

## 为什么需要泛型

考虑一个简单的例子：实现一个返回传入值的函数。

```ts
// 不使用泛型 - 需要为每种类型写一个函数
function returnNumber(arg: number): number {
  return arg;
}

function returnString(arg: string): string {
  return arg;
}

// 使用 any - 丢失类型信息
function returnAny(arg: any): any {
  return arg;
}

const result = returnAny(123); // result 类型是 any，失去了类型检查
```

使用泛型可以保留类型信息：

```ts
function identity<T>(arg: T): T {
  return arg;
}

const num = identity(123);    // num 类型是 number
const str = identity("hello"); // str 类型是 string
```

## 泛型函数

### 基本语法

```ts
// 函数声明
function identity<T>(arg: T): T {
  return arg;
}

// 函数表达式
const identity2 = <T>(arg: T): T => {
  return arg;
};

// 在 TSX 文件中，使用 <T,> 避免与 JSX 混淆
const identity3 = <T,>(arg: T): T => {
  return arg;
};
```

### 调用方式

```ts
// 类型推断 - 编译器自动推断
identity(123);        // T 推断为 number
identity("hello");    // T 推断为 string

// 显式指定类型
identity<number>(123);
identity<string>("hello");
```

### 多类型参数

```ts
function pair<K, V>(key: K, value: V): [K, V] {
  return [key, value];
}

const p1 = pair("name", "Tom");           // [string, string]
const p2 = pair(1, { id: 100 });          // [number, { id: number }]
const p3 = pair<number, string>(1, "a");  // 显式指定
```

## 泛型接口

```ts
// 定义泛型接口
interface Container<T> {
  value: T;
  getValue(): T;
  setValue(value: T): void;
}

// 实现
class Box<T> implements Container<T> {
  value: T;

  constructor(value: T) {
    this.value = value;
  }

  getValue(): T {
    return this.value;
  }

  setValue(value: T): void {
    this.value = value;
  }
}

const numberBox = new Box(123);
const stringBox = new Box<string>("hello");
```

### 泛型接口的两种形式

```ts
// 形式1：泛型作为整个接口的类型参数
interface Response<T> {
  data: T;
  status: number;
  message: string;
}

const userResponse: Response<User> = { ... };

// 形式2：泛型作为函数成员的类型参数
interface Handler {
  <T>(data: T): void;
}

const handler: Handler = <T>(data: T) => {
  console.log(data);
};
```

## 泛型类

```ts
class Stack<T> {
  private items: T[] = [];

  push(item: T): void {
    this.items.push(item);
  }

  pop(): T | undefined {
    return this.items.pop();
  }

  peek(): T | undefined {
    return this.items[this.items.length - 1];
  }

  get size(): number {
    return this.items.length;
  }
}

const numberStack = new Stack<number>();
numberStack.push(1);
numberStack.push(2);
console.log(numberStack.pop()); // 2

const stringStack = new Stack<string>();
stringStack.push("a");
stringStack.push("b");
```

## 泛型约束

默认情况下，泛型类型参数可以代表任何类型，这限制了我们能对其进行的操作。

```ts
function getLength<T>(arg: T): number {
  // ❌ 错误：类型 T 上不存在属性 length
  return arg.length;
}
```

### 使用 extends 约束

```ts
interface Lengthwise {
  length: number;
}

function getLength<T extends Lengthwise>(arg: T): number {
  return arg.length; // ✅ 现在可以访问 length
}

getLength("hello");      // ✅ string 有 length
getLength([1, 2, 3]);    // ✅ array 有 length
getLength({ length: 5 }); // ✅ 对象有 length 属性
// getLength(123);       // ❌ number 没有 length
```

### 使用类型参数约束另一个类型参数

```ts
// K 必须是 T 的键
function getProperty<T, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key];
}

const user = {
  name: "Tom",
  age: 30,
  email: "tom@example.com"
};

const name = getProperty(user, "name");    // string
const age = getProperty(user, "age");      // number
// getProperty(user, "address");  // ❌ 错误：不是 user 的键
```

### 构造函数约束

```ts
interface IConstructor<T> {
  new (...args: any[]): T;
}

function createInstance<T>(ctor: IConstructor<T>, ...args: any[]): T {
  return new ctor(...args);
}

class Person {
  constructor(public name: string) {}
}

const person = createInstance(Person, "Tom");
```

## 泛型默认值

```ts
// 类似函数参数默认值
interface ApiResponse<T = unknown> {
  data: T;
  status: number;
}

const response1: ApiResponse = { data: "hello", status: 200 };     // T = unknown
const response2: ApiResponse<User> = { data: user, status: 200 }; // T = User

// 多个默认值
interface Query<TData = unknown, TError = Error> {
  data: TData;
  error: TError | null;
}
```

## 条件类型

条件类型类似于三元表达式，根据类型关系选择不同的类型分支。

### 基本语法

```ts
type IsString<T> = T extends string ? true : false;

type A = IsString<string>;  // true
type B = IsString<number>;  // false
```

### 实用示例

```ts
// 非空类型
type NonNullable<T> = T extends null | undefined ? never : T;

type C = NonNullable<string | null>;  // string
type D = NonNullable<number | undefined>; // number

// 返回类型提取
type GetReturnType<T> = T extends (...args: any[]) => infer R ? R : never;

type E = GetReturnType<() => string>;  // string
type F = GetReturnType<(x: number) => boolean>; // boolean

// 数组元素类型
type ElementType<T> = T extends (infer E)[] ? E : never;

type G = ElementType<string[]>;  // string
type H = ElementType<number[]>;  // number
```

## infer 关键字

`infer` 用于在条件类型中推断类型，类似于声明一个类型变量。

```ts
// 提取函数参数类型
type Parameters<T> = T extends (...args: infer P) => any ? P : never;

type Fn = (a: string, b: number) => void;
type P = Parameters<Fn>;  // [string, number]

// 提取 Promise 返回类型
type Awaited<T> = T extends Promise<infer U> ? U : T;

type I = Awaited<Promise<string>>;  // string
type J = Awaited<string>;           // string (非 Promise 则原样返回)
```

### 嵌套推断

```ts
// 提取对象中所有函数的返回类型
type ExtractReturnTypes<T> = {
  [K in keyof T]: T[K] extends (...args: any[]) => infer R ? R : never;
};

const api = {
  getUser: () => ({ name: "Tom", age: 30 }),
  getPosts: () => [{ id: 1, title: "Hello" }],
};

type ApiReturns = ExtractReturnTypes<typeof api>;
// { getUser: { name: string; age: number }; getPosts: { id: number; title: string }[] }
```

## 与 Java/C# 泛型的差异

### 类型擦除程度不同

**Java**：泛型在运行时被完全擦除，`List<String>` 在运行时只是 `List`

**TypeScript**：同样擦除，但在编译时提供更强的类型推断

```ts
// TypeScript 编译前
function identity<T>(arg: T): T {
  return arg;
}

// TypeScript 编译后
function identity(arg) {
  return arg;
}
```

### 结构化类型 vs 名义类型

**Java/C#**：名义类型，两个类即使结构相同也不兼容

```java
// Java
class Person { String name; }
class User { String name; }

Person p = new User(); // ❌ 编译错误
```

**TypeScript**：结构化类型（鸭子类型）

```ts
interface Person { name: string; }
interface User { name: string; }

const p: Person = { name: "Tom" };
const u: User = p; // ✅ 结构相同，类型兼容
```

### 类型参数约束

**C#**：支持值类型约束、构造函数约束

```csharp
// C#
where T : class        // 引用类型
where T : struct       // 值类型
where T : new()        // 无参构造函数
where T : BaseClass    // 继承约束
```

**TypeScript**：约束更灵活但类型信息较弱

```ts
// TypeScript
<T extends object>     // 对象类型
<T extends { new(): T }> // 构造函数
<T extends keyof X>    // 键约束
```

### 泛型变型

**C#/Java**：支持声明点变型（协变、逆变）

```csharp
// C#
interface IConsumer<in T> { }    // 逆变
interface IProducer<out T> { }  // 协变
```

**TypeScript**：变型是结构性的，由类型系统自动推断

```ts
// TypeScript 自动处理变型
interface Producer<T> {
  produce(): T;
}

let stringProducer: Producer<string>;
let objectProducer: Producer<object>;
// stringProducer = objectProducer; // ❌ 不允许
// objectProducer = stringProducer; // ✅ 协变，允许
```

## 最佳实践

### 1. 命名约定

```ts
// ✅ 推荐：使用有意义的名称
interface ApiResponse<TData> { }
function getItems<TItem>(items: TItem[]) { }
class Container<TValue> { }

// 也可以，但在简单场景
function identity<T>(arg: T) { }

// ❌ 避免：过度缩写
function process<X>(x: X) { }
```

### 2. 约束尽可能精确

```ts
// ❌ 过于宽松
function getProperty<T>(obj: T, key: string): any {
  return (obj as any)[key];
}

// ✅ 精确约束
function getProperty<T, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key];
}
```

### 3. 使用泛型默认值提高可用性

```ts
interface Result<T = unknown, E = Error> {
  success: boolean;
  data?: T;
  error?: E;
}

// 简单使用
const r1: Result = { success: true, data: "hello" };

// 完整类型
const r2: Result<User, ValidationError> = { ... };
```

### 4. 避免泛型滥用

```ts
// ❌ 不必要的泛型
function getId<T extends { id: string }>(obj: T): string {
  return obj.id;
}

// ✅ 直接使用约束类型
function getId(obj: { id: string }): string {
  return obj.id;
}

// ✅ 泛型在返回类型保留时才有意义
function wrap<T>(value: T): { value: T } {
  return { value };
}
```

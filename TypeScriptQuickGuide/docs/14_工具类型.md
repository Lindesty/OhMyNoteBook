# 工具类型

工具类型（Utility Types）是 TypeScript 内置的类型转换工具，可以快速创建新类型。对于熟悉 Java/C# 的开发者，这些类型类似于 Lombok 或 AutoMapper 的功能，但作用于类型系统层面。

## Partial<T>

将类型 T 的所有属性变为可选。

```ts
interface User {
  id: number;
  name: string;
  email: string;
  age: number;
}

// 手动实现
type PartialUser = {
  id?: number;
  name?: string;
  email?: string;
  age?: number;
};

// 使用 Partial
type PartialUser = Partial<User>;
```

### 实际应用

```ts
interface User {
  id: number;
  name: string;
  email: string;
}

// 更新用户信息时，只更新提供的字段
function updateUser(id: number, updates: Partial<User>): User {
  const user = getUserById(id); // 假设从数据库获取
  return { ...user, ...updates };
}

// 使用
updateUser(1, { name: "New Name" });
updateUser(2, { email: "new@email.com", age: 25 }); // 只更新部分字段
```

## Required<T>

将类型 T 的所有属性变为必需。

```ts
interface User {
  id: number;
  name?: string;
  email?: string;
}

// 所有属性都变成必需
type RequiredUser = Required<User>;
// { id: number; name: string; email: string; }

const user: RequiredUser = {
  id: 1,
  name: "Tom",
  email: "tom@example.com"
};
```

## Readonly<T>

将类型 T 的所有属性变为只读。

```ts
interface User {
  id: number;
  name: string;
}

type ReadonlyUser = Readonly<User>;

const user: ReadonlyUser = {
  id: 1,
  name: "Tom"
};

// user.id = 2;    // ❌ 只读属性不能修改
// user.name = ""; // ❌ 只读属性不能修改
```

### 深层只读

`Readonly` 只作用于第一层，需要递归实现深层只读：

```ts
type DeepReadonly<T> = {
  readonly [K in keyof T]: T[K] extends object
    ? DeepReadonly<T[K]>
    : T[K];
};

interface Config {
  server: {
    host: string;
    port: number;
  };
  database: {
    url: string;
    credentials: {
      username: string;
      password: string;
    };
  };
}

const config: DeepReadonly<Config> = {
  server: { host: "localhost", port: 3000 },
  database: {
    url: "mongodb://localhost",
    credentials: { username: "admin", password: "secret" }
  }
};

// config.server.host = "127.0.0.1"; // ❌ 深层只读
```

## Record<K, T>

创建一个对象类型，键是 K，值是 T。

```ts
// 手动实现
type UserMap = {
  [key: string]: User;
};

// 使用 Record
type UserMap = Record<string, User>;

// 使用
const users: Record<string, User> = {
  "user-1": { id: 1, name: "Tom" },
  "user-2": { id: 2, name: "Jerry" }
};
```

### 实际应用

```ts
// 定义状态映射
type Status = "pending" | "approved" | "rejected";
type StatusLabels = Record<Status, string>;

const statusLabels: StatusLabels = {
  pending: "待审核",
  approved: "已通过",
  rejected: "已拒绝"
};

// 定义事件处理器
type Events = "click" | "hover" | "focus";
type EventHandler = () => void;
type EventHandlers = Record<Events, EventHandler>;

const handlers: EventHandlers = {
  click: () => console.log("clicked"),
  hover: () => console.log("hovered"),
  focus: () => console.log("focused")
};
```

## Pick<T, K>

从类型 T 中选择部分属性 K。

```ts
interface User {
  id: number;
  name: string;
  email: string;
  password: string;
  createdAt: Date;
}

// 只选择 id 和 name
type UserPreview = Pick<User, "id" | "name">;
// { id: number; name: string; }

const preview: UserPreview = {
  id: 1,
  name: "Tom"
};
```

### 实际应用

```ts
interface User {
  id: number;
  name: string;
  email: string;
  password: string;
}

// API 返回时不包含敏感字段
type PublicUser = Pick<User, "id" | "name" | "email">;

function getUser(id: number): PublicUser {
  const user = fetchUser(id);
  const { password, ...publicUser } = user;
  return publicUser;
}
```

## Omit<T, K>

从类型 T 中排除部分属性 K。

```ts
interface User {
  id: number;
  name: string;
  email: string;
  password: string;
}

// 排除 password
type SafeUser = Omit<User, "password">;
// { id: number; name: string; email: string; }

// 排除多个属性
type UserBase = Omit<User, "password" | "createdAt">;
```

### 创建表单数据类型

```ts
interface User {
  id: number;
  name: string;
  email: string;
  createdAt: Date;
  updatedAt: Date;
}

// 创建用户时不包含自动生成的字段
type CreateUserData = Omit<User, "id" | "createdAt" | "updatedAt">;

// 更新用户时不包含不可修改的字段
type UpdateUserData = Omit<User, "id" | "createdAt">;

function createUser(data: CreateUserData): User {
  return {
    ...data,
    id: generateId(),
    createdAt: new Date(),
    updatedAt: new Date()
  };
}
```

## Exclude<T, U>

从类型 T 中排除可以赋值给 U 的类型。

```ts
type T = "a" | "b" | "c" | "d";
type U = "a" | "c";

type Result = Exclude<T, U>; // "b" | "d"

// 实际应用
type AllEvents = "click" | "hover" | "focus" | "blur";
type MouseEvent = Exclude<AllEvents, "focus" | "blur">; // "click" | "hover"
```

## Extract<T, U>

从类型 T 中提取可以赋值给 U 的类型。

```ts
type T = "a" | "b" | "c" | "d";
type U = "a" | "c";

type Result = Extract<T, U>; // "a" | "c"

// 实际应用
type AllEvents = "click" | "hover" | "focus" | "blur";
type FocusEvent = Extract<AllEvents, "focus" | "blur">; // "focus" | "blur"
```

### 从联合类型中提取特定类型

```ts
type User = { type: "user"; id: number };
type Admin = { type: "admin"; id: number; permissions: string[] };
type Guest = { type: "guest" };

type Account = User | Admin | Guest;

// 提取有 id 的类型
type WithId = Extract<Account, { id: number }>;
// User | Admin
```

## NonNullable<T>

从类型 T 中排除 null 和 undefined。

```ts
type T = string | number | null | undefined;

type Result = NonNullable<T>; // string | number

// 实际应用
function process(value: string | null | undefined) {
  const v: NonNullable<typeof value> = value!; // 需要确保非空
  console.log(v.toUpperCase());
}

// 结合类型守卫
function getLength(str: string | null | undefined): number {
  if (str !== null && str !== undefined) {
    return (str as NonNullable<typeof str>).length;
  }
  return 0;
}
```

## ReturnType<T>

获取函数类型 T 的返回类型。

```ts
function getUser() {
  return {
    id: 1,
    name: "Tom",
    email: "tom@example.com"
  };
}

type User = ReturnType<typeof getUser>;
// { id: number; name: string; email: string; }

// 实际应用
function fetchUser(id: number) {
  return fetch(`/api/users/${id}`).then(res => res.json());
}

type UserResponse = ReturnType<typeof fetchUser>; // Promise<any>

// 结合泛型
function createState<T>(initialValue: T) {
  let value = initialValue;
  return {
    get: () => value,
    set: (newValue: T) => { value = newValue; }
  };
}

type State<T> = ReturnType<typeof createState<T>>;
```

## Parameters<T>

获取函数类型 T 的参数类型，返回元组。

```ts
function createUser(name: string, age: number, email: string) {
  return { name, age, email };
}

type UserParams = Parameters<typeof createUser>;
// [string, number, string]

// 使用场景：包装函数
function withLogging<T extends (...args: any[]) => any>(fn: T) {
  return (...args: Parameters<T>): ReturnType<T> => {
    console.log("Calling with args:", args);
    const result = fn(...args);
    console.log("Result:", result);
    return result;
  };
}
```

## InstanceType<T>

获取构造函数类型 T 的实例类型。

```ts
class User {
  constructor(
    public name: string,
    public age: number
  ) {}

  greet() {
    return `Hello, ${this.name}`;
  }
}

type UserInstance = InstanceType<typeof User>;
// User

// 实际应用
function createInstance<T extends new (...args: any[]) => any>(
  ctor: T,
  ...args: ConstructorParameters<T>
): InstanceType<T> {
  return new ctor(...args);
}

const user = createInstance(User, "Tom", 30);
// user: User
```

## ConstructorParameters<T>

获取构造函数类型 T 的参数类型。

```ts
class User {
  constructor(
    public name: string,
    public age: number,
    public email: string
  ) {}
}

type UserParams = ConstructorParameters<typeof User>;
// [string, number, string]
```

## 映射类型（Mapped Types）

映射类型允许基于旧类型创建新类型。

### 基本语法

```ts
type Mapped<T> = {
  [K in keyof T]: T[K];
};

interface User {
  id: number;
  name: string;
}

type UserCopy = Mapped<User>; // 与 User 相同
```

### 修饰符

```ts
// 添加 readonly
type Readonly<T> = {
  readonly [K in keyof T]: T[K];
};

// 添加可选
type Optional<T> = {
  [K in keyof T]?: T[K];
};

// 移除 readonly
type Mutable<T> = {
  -readonly [K in keyof T]: T[K];
};

// 移除可选
type Required<T> = {
  [K in keyof T]-?: T[K];
};
```

### 重命名键

```ts
type AppendGetter<T> = {
  [K in keyof T as `get${Capitalize<string & K>}`]: () => T[K];
};

interface User {
  name: string;
  age: number;
}

type UserGetters = AppendGetter<User>;
// { getName: () => string; getAge: () => number; }
```

### 过滤属性

```ts
type OnlyString<T> = {
  [K in keyof T as T[K] extends string ? K : never]: T[K];
};

interface User {
  id: number;
  name: string;
  email: string;
  age: number;
}

type StringFields = OnlyString<User>;
// { name: string; email: string; }
```

## 自定义工具类型

### DeepPartial

```ts
type DeepPartial<T> = {
  [K in keyof T]?: T[K] extends object
    ? DeepPartial<T[K]>
    : T[K];
};

interface Config {
  server: {
    host: string;
    port: number;
  };
  database: {
    url: string;
    credentials: {
      username: string;
      password: string;
    };
  };
}

const config: DeepPartial<Config> = {
  server: {
    host: "localhost"
    // port 可选
  }
};
```

### DeepRequired

```ts
type DeepRequired<T> = {
  [K in keyof T]-?: T[K] extends object
    ? DeepRequired<T[K]>
    : T[K];
};
```

### PickByType

```ts
type PickByType<T, U> = {
  [K in keyof T as T[K] extends U ? K : never]: T[K];
};

interface User {
  id: number;
  name: string;
  email: string;
  age: number;
  active: boolean;
}

type StringFields = PickByType<User, string>;
// { name: string; email: string; }

type NumberFields = PickByType<User, number>;
// { id: number; age: number; }
```

### OmitByType

```ts
type OmitByType<T, U> = {
  [K in keyof T as T[K] extends U ? never : K]: T[K];
};

type NonStringFields = OmitByType<User, string>;
// { id: number; age: number; active: boolean; }
```

### Merge

```ts
type Merge<T, U> = Omit<T, keyof U> & U;

interface Defaults {
  host: string;
  port: number;
  timeout: number;
}

interface Config {
  host: string;
  port: number;
  debug: boolean;
}

type Merged = Merge<Defaults, Config>;
// { timeout: number; host: string; port: number; debug: boolean; }
```

### RequiredKeys

```ts
type RequiredKeys<T> = {
  [K in keyof T]-?: {} extends Pick<T, K> ? never : K;
}[keyof T];

interface User {
  id: number;      // required
  name: string;    // required
  email?: string;   // optional
  age?: number;     // optional
}

type RequiredFields = RequiredKeys<User>; // "id" | "name"
```

### OptionalKeys

```ts
type OptionalKeys<T> = {
  [K in keyof T]-?: {} extends Pick<T, K> ? K : never;
}[keyof T];

type OptionalFields = OptionalKeys<User>; // "email" | "age"
```

### GetPath

```ts
type GetPath<T, P extends string> =
  P extends `${infer K}.${infer Rest}`
    ? K extends keyof T
      ? GetPath<T[K], Rest>
      : never
    : P extends keyof T
    ? T[P]
    : never;

interface User {
  profile: {
    name: string;
    address: {
      city: string;
      country: string;
    };
  };
}

type City = GetPath<User, "profile.address.city">; // string
type Name = GetPath<User, "profile.name">; // string
```

## 工具类型速查表

| 工具类型 | 作用 | 示例 |
|---------|------|------|
| `Partial<T>` | 所有属性可选 | `Partial<User>` |
| `Required<T>` | 所有属性必需 | `Required<User>` |
| `Readonly<T>` | 所有属性只读 | `Readonly<User>` |
| `Record<K, T>` | 创建键值对类型 | `Record<string, User>` |
| `Pick<T, K>` | 选择部分属性 | `Pick<User, "id" \| "name">` |
| `Omit<T, K>` | 排除部分属性 | `Omit<User, "password">` |
| `Exclude<T, U>` | 从联合类型排除 | `Exclude<"a"\|"b"\|"c", "a">` |
| `Extract<T, U>` | 从联合类型提取 | `Extract<"a"\|"b"\|"c", "a"\|"d">` |
| `NonNullable<T>` | 排除 null/undefined | `NonNullable<string\|null>` |
| `ReturnType<T>` | 获取函数返回类型 | `ReturnType<typeof fn>` |
| `Parameters<T>` | 获取函数参数类型 | `Parameters<typeof fn>` |
| `InstanceType<T>` | 获取实例类型 | `InstanceType<typeof Class>` |
| `ConstructorParameters<T>` | 获取构造函数参数 | `ConstructorParameters<typeof Class>` |

## 与 Java/C# 的对比

| 功能 | Java | C# | TypeScript |
|------|------|-----|------------|
| 不可变类型 | 需要 immutable 库 | `init` 属性 | `Readonly<T>` |
| Builder 模式 | 手动或 Lombok | 手动或库支持 | `Partial<T>` |
| 对象映射 | AutoMapper | AutoMapper | `Pick`/`Omit` |
| 动态属性 | Map | Dictionary | `Record<K, T>` |
| 类型转换 | 手动 | 手动 | 工具类型链式调用 |

## 最佳实践

### 1. 组合使用工具类型

```ts
interface User {
  id: number;
  name: string;
  email: string;
  password: string;
  createdAt: Date;
}

// 创建用户时的数据（排除自动生成字段）
type CreateUserDTO = Omit<User, "id" | "createdAt">;

// 更新用户时的数据（所有字段可选）
type UpdateUserDTO = Partial<CreateUserDTO>;

// API 返回的用户数据（排除敏感字段）
type UserResponse = Omit<User, "password">;
```

### 2. 使用泛型约束保持灵活性

```ts
function updateEntity<T extends { id: number }>(
  entity: T,
  updates: Partial<T>
): T {
  return { ...entity, ...updates };
}
```

### 3. 创建项目专用工具类型

```ts
// 在项目中复用
type ID = string | number;
type Timestamp = number | Date;

interface BaseEntity {
  id: ID;
  createdAt: Timestamp;
  updatedAt: Timestamp;
}

type CreateInput<T> = Omit<T, "id" | "createdAt" | "updatedAt">;
type UpdateInput<T> = Partial<CreateInput<T>>;
```

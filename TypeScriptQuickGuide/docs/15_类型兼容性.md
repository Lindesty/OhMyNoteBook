# 类型兼容性

TypeScript 使用结构化类型系统（Structural Type System），也称为鸭子类型（Duck Typing）。这与 Java/C# 的名义类型系统（Nominal Type System）有本质区别。理解类型兼容性对于正确使用 TypeScript 至关重要。

## 结构化类型 vs 名义类型

### 名义类型（Java/C#）

两个类型是否兼容取决于它们的名称：

```java
// Java
class Person { String name; }
class User { String name; }

Person p = new User(); // ❌ 编译错误：类型不兼容
```

### 结构化类型（TypeScript）

两个类型是否兼容取决于它们的结构：

```ts
interface Person { name: string; }
interface User { name: string; }

const p: Person = { name: "Tom" };
const u: User = p; // ✅ 编译通过：结构相同
```

这意味着只要有相同的属性和方法，类型就兼容：

```ts
interface Point2D { x: number; y: number; }
interface Point3D { x: number; y: number; z: number; }

const point3D: Point3D = { x: 1, y: 2, z: 3 };
const point2D: Point2D = point3D; // ✅ Point3D 有 Point2D 的所有属性

// 反过来不行
// const point3D: Point3D = point2D; // ❌ 缺少 z 属性
```

## 基本规则

### 子类型赋值

属性更多的类型可以赋值给属性更少的类型：

```ts
interface Animal {
  name: string;
}

interface Dog extends Animal {
  breed: string;
}

const dog: Dog = { name: "Buddy", breed: "Labrador" };
const animal: Animal = dog; // ✅ Dog 有 Animal 的所有属性
```

### 对象字面量

对象字面量有更严格的检查（多余属性检查）：

```ts
interface User {
  name: string;
}

const user: User = { name: "Tom" }; // ✅

// ❌ 多余属性检查
const user2: User = { name: "Tom", age: 30 }; // 错误

// ✅ 绕过检查的方式
const obj = { name: "Tom", age: 30 };
const user3: User = obj; // 通过中间变量
```

### 函数参数

函数参数类型必须匹配或更宽松：

```ts
let fn1 = (x: string) => x;
let fn2 = (x: string | number) => x;

fn1 = fn2; // ✅ fn2 接受更广泛的类型
// fn2 = fn1; // ❌ fn1 只接受 string
```

## 协变与逆变

### 协变（Covariance）

子类型可以赋值给父类型：

```ts
interface Animal { name: string; }
interface Dog extends Animal { breed: string; }

// 协变：Dog[] 可以赋值给 Animal[]
let dogs: Dog[] = [{ name: "Buddy", breed: "Labrador" }];
let animals: Animal[] = dogs; // ✅ 协变

// 但要注意：这可能导致运行时问题
animals.push({ name: "Cat" }); // 编译通过！
// dogs[1] 现在是 { name: "Cat" }，但 Dog 需要 breed
```

### 逆变（Contravariance）

函数参数类型相反：

```ts
interface Animal { name: string; }
interface Dog extends Animal { breed: string; }

// 动物处理函数
let handleAnimal = (animal: Animal) => console.log(animal.name);
// 狗处理函数
let handleDog = (dog: Dog) => console.log(dog.breed);

// 逆变：handleAnimal 可以赋值给 handleDog 类型的变量
let handler: (dog: Dog) => void = handleAnimal; // ✅ 逆变

// 为什么？
// handler 期望接收 Dog
// handleAnimal 可以处理任何 Animal，包括 Dog
// 所以把 handleAnimal 赋给 handler 是安全的

// 反过来不安全
// let handler2: (animal: Animal) => void = handleDog; // ❌
// handler2 可能传入任意 Animal
// 但 handleDog 需要 Dog 的 breed 属性
```

### 双向协变

TypeScript 默认允许函数参数双向协变（为了灵活性）：

```ts
interface Animal { name: string; }
interface Dog extends Animal { breed: string; }

type AnimalHandler = (animal: Animal) => void;
type DogHandler = (dog: Dog) => void;

let handleAnimal: AnimalHandler = (a) => console.log(a.name);
let handleDog: DogHandler = (d) => console.log(d.breed);

// 双向协变（默认行为）
let h1: AnimalHandler = handleDog; // ✅ 技术上允许（但可能不安全）
let h2: DogHandler = handleAnimal; // ✅ 安全
```

启用 `strictFunctionTypes` 可以禁止双向协变：

```json
{
  "compilerOptions": {
    "strictFunctionTypes": true
  }
}
```

## 函数兼容性

### 参数数量

目标函数的参数可以更少：

```ts
let fn1: (a: number, b: number) => number = (a, b) => a + b;
let fn2: (a: number) => number = fn1; // ✅ 忽略多余参数

// 实际使用
[1, 2, 3].map((x, index, array) => x * 2);
// 等价于
[1, 2, 3].map(x => x * 2); // ✅ 忽略 index 和 array
```

### 返回类型

返回类型必须是子类型：

```ts
interface Animal { name: string; }
interface Dog extends Animal { breed: string; }

let createAnimal = (): Animal => ({ name: "Animal" });
let createDog = (): Dog => ({ name: "Dog", breed: "Labrador" });

let factory1: () => Animal = createDog; // ✅ Dog 是 Animal 的子类型
// let factory2: () => Dog = createAnimal; // ❌ Animal 不是 Dog 的子类型
```

## 类兼容性

### 实例成员

只检查公共实例成员：

```ts
class Animal {
  name: string = "";
  private secret: string = ""; // 私有成员不参与比较
}

class Dog {
  name: string = "";
  private secret: string = ""; // 私有成员不参与比较
}

let animal: Animal = new Dog(); // ✅ 结构相同

// 但是如果有相同来源的私有成员
class Cat extends Animal {}
let cat: Animal = new Cat(); // ✅
```

### protected 成员

```ts
class Parent {
  protected name: string = "";
}

class Child extends Parent {
  public getName() { return this.name; }
}

class Other {
  protected name: string = "";
}

let parent: Parent = new Child(); // ✅
// let other: Parent = new Other(); // ❌ protected 成员来源不同
```

## 泛型兼容性

### 类型参数不影响结构

```ts
interface Empty<T> {}

let e1: Empty<string> = {};
let e2: Empty<number> = {};

e1 = e2; // ✅ 结构相同，类型参数不影响
```

### 使用类型参数时

```ts
interface Container<T> {
  value: T;
}

let c1: Container<string> = { value: "hello" };
let c2: Container<number> = { value: 123 };

// c1 = c2; // ❌ 类型不同
```

### 泛型函数

```ts
function identity<T>(arg: T): T {
  return arg;
}

let fn1: <T>(arg: T) => T = identity;
let fn2: <U>(arg: U) => U = identity; // ✅ 类型参数名称不重要
```

## 枚举兼容性

### 数字枚举

数字枚举与 number 兼容：

```ts
enum Status {
  Pending,
  Active,
  Completed
}

let s: Status = Status.Active;
let n: number = s; // ✅ 枚举可以赋给 number

let num: number = 5;
// let status: Status = num; // ❌ number 不能赋给枚举（严格模式）
```

### 字符串枚举

字符串枚举与 string 不兼容：

```ts
enum Direction {
  Up = "UP",
  Down = "DOWN"
}

let d: Direction = Direction.Up;
// let s: string = d; // ❌ 字符串枚举与 string 不兼容
```

### 混合枚举

```ts
enum Mixed {
  A = 1,
  B = "hello"
}

let m1: Mixed = Mixed.A; // number
let m2: Mixed = Mixed.B; // string
```

## 可空类型

### strictNullChecks 关闭

```ts
let name: string = null; // ✅ 允许
let age: number = undefined; // ✅ 允许
```

### strictNullChecks 开启

```ts
let name: string = null; // ❌ 错误
let age: number | null = null; // ✅ 必须显式声明
```

### 可选属性

```ts
interface User {
  name: string;
  email?: string; // email: string | undefined
}

const user: User = { name: "Tom" };
console.log(user.email); // string | undefined
```

## any 和 unknown

### any

关闭类型检查：

```ts
let value: any = "hello";
value = 123;
value = { a: 1 };

let str: string = value; // ✅ any 兼容任何类型
value.nonExistent(); // ✅ 编译通过（运行时错误）
```

### unknown

类型安全的 any：

```ts
let value: unknown = "hello";
value = 123;

// value.toUpperCase(); // ❌ 不能直接使用
// let str: string = value; // ❌ 不能直接赋值

// 必须先进行类型检查
if (typeof value === "string") {
  console.log(value.toUpperCase()); // ✅
}

// 或使用类型断言
let str = value as string;
```

### never

never 是所有类型的子类型：

```ts
function fail(message: string): never {
  throw new Error(message);
}

let str: string = fail("error"); // ✅ never 是 string 的子类型
```

## 类型断言的影响

类型断言可以绕过兼容性检查：

```ts
interface User {
  name: string;
  email: string;
}

const obj = { name: "Tom" } as User; // ✅ 编译通过
// 但运行时 email 是 undefined

// 双重断言（危险）
const str = "hello" as unknown as number;
// console.log(str.toFixed()); // 运行时错误
```

## 实践中的问题

### API 响应类型

```ts
interface ApiResponse {
  data: {
    id: number;
    name: string;
  };
}

// ❌ 不安全：API 返回的结构可能不完全匹配
const response = JSON.parse(json) as ApiResponse;

// ✅ 安全：使用类型守卫验证
function isApiResponse(value: unknown): value is ApiResponse {
  const data = value as ApiResponse;
  return (
    typeof value === "object" &&
    value !== null &&
    "data" in value &&
    typeof data.data.id === "number" &&
    typeof data.data.name === "string"
  );
}
```

### 回调函数类型

```ts
// DOM 事件处理
document.addEventListener("click", (event: MouseEvent) => {
  // ...
});

// 可以使用更通用的类型
document.addEventListener("click", (event: Event) => {
  // Event 是 MouseEvent 的父类型
  // 这里我们"丢失"了 MouseEvent 特有的属性
});

// React 事件
const handleChange = (e: ChangeEvent<HTMLInputElement>) => {
  console.log(e.target.value);
};

// 可以传递给期望更通用类型的函数
const fn: (e: Event) => void = handleChange; // ✅
```

## 与 Java/C# 的对比

| 特性 | Java | C# | TypeScript |
|------|------|-----|------------|
| 类型系统 | 名义类型 | 名义类型 | 结构化类型 |
| 类型兼容 | 继承/实现关系 | 继承/实现关系 | 结构相同即可 |
| 接口兼容 | 必须显式实现 | 必须显式实现 | 结构匹配即可 |
| 泛型变型 | 声明点变型 | 声明点变型 | 使用点变型 |
| 数组协变 | 不安全但允许 | 不安全但允许 | 不安全但允许 |
| null/undefined | 独立类型 | 独立类型 | 可选类型/联合类型 |

**Java 示例**：
```java
interface User { String getName(); }
class Person implements User { public String getName() { return "Tom"; } }

User user = new Person(); // ✅ 显式实现
// User user2 = new Object() { public String getName() { return "Tom"; } }; // ❌
```

**TypeScript 示例**：
```ts
interface User { getName(): string; }

const person = { getName: () => "Tom" };
const user: User = person; // ✅ 结构匹配，无需显式实现
```

## 最佳实践

### 1. 使用结构化类型的优势

```ts
// 定义最小接口
interface Nameable {
  name: string;
}

function greet(item: Nameable) {
  console.log(`Hello, ${item.name}`);
}

// 任何有 name 属性的对象都可以使用
greet({ name: "Tom" });
greet({ name: "Alice", age: 30 });
greet(new Person("Bob"));
```

### 2. 使用 unknown 而非 any

```ts
// ❌ 使用 any
function process(data: any) {
  return data.value; // 不安全
}

// ✅ 使用 unknown
function process(data: unknown) {
  if (typeof data === "object" && data !== null && "value" in data) {
    return data.value;
  }
  return null;
}
```

### 3. 使用类型守卫保护类型安全

```ts
function isArrayOfStrings(value: unknown): value is string[] {
  return Array.isArray(value) && value.every(item => typeof item === "string");
}
```

### 4. 避免过度使用类型断言

```ts
// ❌ 绕过类型检查
const user = data as User;

// ✅ 验证数据
function validateUser(data: unknown): User | null {
  if (isUser(data)) return data;
  return null;
}
```

### 5. 理解函数参数的双向协变风险

```ts
// 如果启用了 strictFunctionTypes
interface Animal { name: string; }
interface Dog extends Animal { breed: string; }

type AnimalHandler = (animal: Animal) => void;
type DogHandler = (dog: Dog) => void;

// 逆变：安全
let h1: DogHandler = (animal: Animal) => console.log(animal.name);

// 协变：不安全
// let h2: AnimalHandler = (dog: Dog) => console.log(dog.breed);
```

# 装饰器

装饰器是一种特殊的声明，可以附加到类、方法、属性或参数上，以修改其行为或添加元数据。如果你熟悉 Java 的注解（Annotation）或 C# 的特性（Attribute），TypeScript 的装饰器概念类似，但更加灵活和强大。

> **注意**：装饰器是 ECMAScript 标准提案，TypeScript 5.0+ 已支持标准装饰器。本教程基于标准装饰器语法。

## 启用装饰器

在 `tsconfig.json` 中启用：

```json
{
  "compilerOptions": {
    "experimentalDecorators": true,
    "emitDecoratorMetadata": true
  }
}
```

> `experimentalDecorators` 启用旧版装饰器（TypeScript 早期实现）
> TypeScript 5.0+ 默认支持标准装饰器，无需此选项

## 装饰器概念

装饰器本质上是一个函数，在运行时被调用，可以修改或增强被装饰的元素。

```ts
// 装饰器就是一个函数
function log(target: any, context: any) {
  console.log("装饰器被调用");
}

// 使用装饰器
@log
class MyClass {}
```

### 与 Java 注解的对比

| 特性 | Java 注解 | TypeScript 装饰器 |
|------|----------|------------------|
| 本质 | 元数据标记 | 运行时函数 |
| 执行时机 | 编译时/运行时（取决于注解） | 运行时 |
| 能否修改行为 | 需要注解处理器 | 直接修改 |
| 内置注解 | @Override, @Deprecated 等 | 无内置装饰器 |

### 与 C# 特性的对比

| 特性 | C# 特性 | TypeScript 装饰器 |
|------|---------|-------------------|
| 定义方式 | 继承 Attribute 类 | 普通函数 |
| 参数 | 固定构造函数参数 | 灵活参数 |
| 运行时访问 | 反射获取 | 直接执行 |

## 类装饰器

类装饰器应用于类构造函数，可以修改或替换类定义。

### 基本语法

```ts
function sealed(constructor: Function) {
  Object.seal(constructor);
  Object.seal(constructor.prototype);
}

@sealed
class BankAccount {
  balance = 0;
}

// 无法添加新属性
// BankAccount.prototype.newMethod = () => {}; // ❌ 运行时报错
```

### 替换类构造函数

```ts
function logged<T extends { new(...args: any[]): {} }>(constructor: T) {
  return class extends constructor {
    constructor(...args: any[]) {
      console.log(`Creating instance with args: ${JSON.stringify(args)}`);
      super(...args);
      console.log(`Instance created`);
    }
  };
}

@logged
class Person {
  constructor(public name: string, public age: number) {}
}

const person = new Person("Tom", 30);
// 输出:
// Creating instance with args: ["Tom",30]
// Instance created
```

### 装饰器工厂

当需要传递参数时，使用装饰器工厂：

```ts
function ClassDecorator(options: { version: string }) {
  return function <T extends { new(...args: any[]): {} }>(constructor: T) {
    return class extends constructor {
      version = options.version;
    };
  };
}

@ClassDecorator({ version: "1.0.0" })
class ApiService {
  version!: string;
}

const api = new ApiService();
console.log(api.version); // "1.0.0"
```

## 方法装饰器

方法装饰器应用于类的方法，可以修改、替换或包装方法。

### 标准装饰器（TypeScript 5.0+）

```ts
function logged(
  value: Function,
  context: ClassMethodDecoratorContext
) {
  const methodName = String(context.name);

  return function (this: any, ...args: any[]) {
    console.log(`Calling ${methodName} with args:`, args);
    const result = value.call(this, ...args);
    console.log(`${methodName} returned:`, result);
    return result;
  };
}

class Calculator {
  @logged
  add(a: number, b: number): number {
    return a + b;
  }
}

const calc = new Calculator();
calc.add(1, 2);
// 输出:
// Calling add with args: [1, 2]
// add returned: 3
```

### 旧版装饰器语法

```ts
function log(
  target: any,
  propertyKey: string,
  descriptor: PropertyDescriptor
) {
  const original = descriptor.value;

  descriptor.value = function (...args: any[]) {
    console.log(`Calling ${propertyKey} with args:`, args);
    const result = original.apply(this, args);
    console.log(`${propertyKey} returned:`, result);
    return result;
  };

  return descriptor;
}

class Calculator {
  @log
  add(a: number, b: number): number {
    return a + b;
  }
}
```

### 实际应用示例

```ts
// 缓存装饰器
function cache(
  value: Function,
  context: ClassMethodDecoratorContext
) {
  const cacheMap = new Map<string, any>();

  return function (this: any, ...args: any[]) {
    const key = JSON.stringify(args);
    if (cacheMap.has(key)) {
      console.log("Cache hit!");
      return cacheMap.get(key);
    }

    const result = value.call(this, ...args);
    cacheMap.set(key, result);
    return result;
  };
}

class DataFetcher {
  @cache
  fetchData(id: number): string {
    console.log("Fetching from server...");
    return `Data-${id}`;
  }
}

const fetcher = new DataFetcher();
fetcher.fetchData(1); // Fetching from server...
fetcher.fetchData(1); // Cache hit!
```

## 属性装饰器

属性装饰器用于修改或添加属性的行为。

### 标准装饰器

```ts
function uppercase(
  target: undefined,
  context: ClassFieldDecoratorContext
) {
  return function (initialValue: string) {
    return initialValue.toUpperCase();
  };
}

class Person {
  @uppercase
  name = "tom"; // 将被转换为 "TOM"
}

const person = new Person();
console.log(person.name); // "TOM"
```

### 只读属性

```ts
function readonly(
  target: undefined,
  context: ClassFieldDecoratorContext
) {
  context.addInitializer(function (this: any) {
    Object.defineProperty(this, context.name, {
      value: this[context.name],
      writable: false,
      configurable: false
    });
  });
}

class Config {
  @readonly
  API_KEY = "secret-key";
}

const config = new Config();
// config.API_KEY = "new-key"; // ❌ 运行时报错
```

## Getter/Setter 装饰器

```ts
function validate(
  value: (v: any) => any,
  context: ClassSetterDecoratorContext
) {
  return function (this: any, newValue: any) {
    if (newValue < 0) {
      throw new Error("Value cannot be negative");
    }
    return value.call(this, newValue);
  };
}

class BankAccount {
  private _balance = 0;

  get balance() {
    return this._balance;
  }

  @validate
  set balance(value: number) {
    this._balance = value;
  }
}

const account = new BankAccount();
account.balance = 100; // ✅
// account.balance = -50; // ❌ Error: Value cannot be negative
```

## 参数装饰器

参数装饰器用于标记或修改方法的参数。

### 旧版语法

```ts
function Required(
  target: any,
  propertyKey: string | symbol,
  parameterIndex: number
) {
  // 存储需要验证的参数索引
  const requiredParams: number[] =
    Reflect.getOwnMetadata("required", target, propertyKey) || [];
  requiredParams.push(parameterIndex);
  Reflect.defineMetadata("required", requiredParams, target, propertyKey);
}

function validateParams(
  target: any,
  propertyKey: string,
  descriptor: PropertyDescriptor
) {
  const method = descriptor.value;

  descriptor.value = function (...args: any[]) {
    const requiredParams: number[] =
      Reflect.getOwnMetadata("required", target, propertyKey) || [];

    for (const index of requiredParams) {
      if (args[index] === undefined || args[index] === null) {
        throw new Error(`Parameter at index ${index} is required`);
      }
    }

    return method.apply(this, args);
  };
}

class UserService {
  @validateParams
  createUser(@Required name: string, @Required email: string) {
    console.log(`Creating user: ${name}, ${email}`);
  }
}

const service = new UserService();
service.createUser("Tom", "tom@example.com"); // ✅
// service.createUser("Tom", undefined); // ❌ Error
```

> 参数装饰器需要 `reflect-metadata` 库支持

## 装饰器组合与执行顺序

多个装饰器可以应用于同一声明上。

### 应用顺序

```ts
function first() {
  console.log("first(): factory");
  return function (target: any, context: any) {
    console.log("first(): decorator");
  };
}

function second() {
  console.log("second(): factory");
  return function (target: any, context: any) {
    console.log("second(): decorator");
  };
}

class Example {
  @first()
  @second()
  method() {}
}

// 输出:
// first(): factory
// second(): factory
// second(): decorator
// first(): decorator
```

**执行顺序规则**：
1. 工厂函数从上到下执行（收集装饰器）
2. 装饰器函数从下到上执行（应用装饰器）

### 类与方法装饰器

```ts
function classDecorator() {
  console.log("class decorator factory");
  return function (target: any) {
    console.log("class decorator applied");
  };
}

function methodDecorator() {
  console.log("method decorator factory");
  return function (target: any, context: any) {
    console.log("method decorator applied");
  };
}

@classDecorator()
class Example {
  @methodDecorator()
  method() {}
}

// 输出:
// method decorator factory
// class decorator factory
// class decorator applied
// method decorator applied
```

**顺序**：
1. 方法装饰器工厂
2. 类装饰器工厂
3. 类装饰器应用
4. 方法装饰器应用

## 最佳实践

### 1. 保持装饰器简单

```ts
// ❌ 装饰器中包含复杂逻辑
function complexDecorator(target: any) {
  // 几十行复杂逻辑...
}

// ✅ 装饰器只做一件事，复杂逻辑委托出去
function cache(
  value: Function,
  context: ClassMethodDecoratorContext
) {
  return createCachedFunction(value);
}
```

### 2. 使用装饰器工厂增加可配置性

```ts
// ✅ 可配置的装饰器
function Log(options: { level: 'info' | 'debug' | 'error' } = { level: 'info' }) {
  return function (value: Function, context: ClassMethodDecoratorContext) {
    // 使用 options.level
  };
}

class Example {
  @Log({ level: 'debug' })
  method() {}
}
```

### 3. 注意装饰器顺序

```ts
// 执行顺序很重要
class Example {
  @cache     // 最后执行（缓存结果）
  @validate  // 先执行（验证参数）
  @log       // 中间执行（记录日志）
  method() {}
}
```

### 4. 类型安全

```ts
// 使用泛型保持类型安全
function memoize<T extends Function>(
  fn: T,
  context: ClassMethodDecoratorContext
): T {
  const cache = new Map();
  return function (this: any, ...args: any[]) {
    const key = JSON.stringify(args);
    if (cache.has(key)) return cache.get(key);
    const result = fn.call(this, ...args);
    cache.set(key, result);
    return result;
  } as T;
}
```

# 声明文件

声明文件（Declaration Files，`.d.ts`）用于为 JavaScript 代码提供类型信息。当使用没有 TypeScript 类型定义的 JavaScript 库时，需要编写或引入声明文件。对于来自 Java/C# 的开发者，这类似于为无类型库添加类型定义。

## 什么是声明文件

声明文件只包含类型声明，不包含实现代码，编译后不生成 JavaScript。

```ts
// utils.d.ts
declare function greet(name: string): string;
declare const version: string;
declare class Calculator {
  add(a: number, b: number): number;
  subtract(a: number, b: number): number;
}
```

## declare 关键字

`declare` 告诉 TypeScript 某个值存在于运行时，即使它没有在 TypeScript 代码中定义。

### declare var / let / const

```ts
// 声明全局变量
declare var __DEV__: boolean;
declare let globalConfig: { api: string };
declare const API_KEY: string;

// 使用
if (__DEV__) {
  console.log("Development mode");
}
console.log(API_KEY);
```

### declare function

```ts
// 声明全局函数
declare function require(module: string): unknown;
declare function alert(message: string): void;

// 使用
const fs = require("fs");
alert("Hello!");
```

### declare class

```ts
// 声明全局类
declare class Observable<T> {
  constructor(subscribe: (observer: Observer<T>) => void);
  subscribe(observer: Observer<T>): Subscription;
  pipe<R>(...operators: OperatorFunction<T, R>[]): Observable<R>;
}

interface Observer<T> {
  next(value: T): void;
  error(err: unknown): void;
  complete(): void;
}
```

### declare namespace

```ts
// 声明全局命名空间
declare namespace NodeJS {
  interface ProcessEnv {
    NODE_ENV: "development" | "production";
    API_URL: string;
    [key: string]: string | undefined;
  }
}

// 使用
process.env.NODE_ENV;
process.env.API_URL;
```

## 全局声明

当变量/函数在全局作用域可用时，使用全局声明。

### 示例：为 jQuery 添加声明

```ts
// jquery.d.ts
declare const $: JQueryStatic;

interface JQueryStatic {
  (selector: string): JQuery;
  ajax(settings: JQueryAjaxSettings): JQueryXHR;
}

interface JQuery {
  html(): string;
  html(html: string): JQuery;
  on(event: string, handler: (e: Event) => void): JQuery;
}

interface JQueryAjaxSettings {
  url: string;
  method?: string;
  data?: unknown;
  success?: (data: any) => void;
  error?: (xhr: JQueryXHR) => void;
}

interface JQueryXHR {
  responseText: string;
  status: number;
}
```

## 模块声明

当代码作为模块导出时，使用模块声明。

### 声明模块

```ts
// types/my-lib.d.ts

// 方式1：声明整个模块
declare module "my-lib" {
  export function greet(name: string): string;
  export class Calculator {
    add(a: number, b: number): number;
  }
  export const version: string;
  export default class MyLib {
    init(): void;
  }
}

// 使用
import MyLib, { greet, Calculator, version } from "my-lib";
```

### 声明文件与源码对应

```ts
// my-utils/index.js (JavaScript 源码)
function greet(name) {
  return `Hello, ${name}!`;
}
function add(a, b) {
  return a + b;
}
module.exports = { greet, add };

// my-utils/index.d.ts (类型声明)
export function greet(name: string): string;
export function add(a: number, b: number): number;
```

### 使用 export =

```ts
// commonjs.d.ts
declare module "legacy-lib" {
  interface LegacyLib {
    doSomething(): void;
  }

  const LegacyLib: LegacyLib;
  export = LegacyLib;
}

// 使用
import LegacyLib = require("legacy-lib");
LegacyLib.doSomething();
```

## 命名空间声明

命名空间用于组织代码，避免全局污染。

### 定义命名空间

```ts
// utils.d.ts
declare namespace Utils {
  interface User {
    id: number;
    name: string;
  }

  function createUser(name: string): User;

  namespace Validation {
    function isEmail(email: string): boolean;
    function isPhone(phone: string): boolean;
  }
}

// 使用
const user: Utils.User = Utils.createUser("Tom");
const valid = Utils.Validation.isEmail("test@example.com");
```

### 命名空间与模块结合

```ts
// my-module.d.ts
export as namespace MyModule;

export interface Config {
  apiUrl: string;
  timeout: number;
}

export function init(config: Config): void;

// 在全局环境下使用
// <script src="my-module.js"></script>
// MyModule.init({ apiUrl: "...", timeout: 5000 });
```

## 为第三方库编写声明

### 场景：为简单的 JS 库添加类型

```javascript
// my-storage.js
const Storage = {
  get(key) {
    return localStorage.getItem(key);
  },
  set(key, value) {
    localStorage.setItem(key, JSON.stringify(value));
  },
  remove(key) {
    localStorage.removeItem(key);
  }
};
module.exports = Storage;
```

```ts
// my-storage.d.ts
declare module "my-storage" {
  interface Storage {
    get<T = string>(key: string): T | null;
    set<T>(key: string, value: T): void;
    remove(key: string): void;
  }

  const storage: Storage;
  export default storage;
}
```

### 场景：为 CDN 引入的库添加类型

```html
<!-- 在 HTML 中通过 script 标签引入 -->
<script src="https://cdn.example.com/analytics.js"></script>
```

```ts
// analytics.d.ts
declare const analytics: {
  track(event: string, data?: Record<string, unknown>): void;
  identify(userId: string, traits?: Record<string, unknown>): void;
  page(category?: string, name?: string): void;
};

// 在 tsconfig.json 中包含此文件
// 或使用三斜线指令
/// <reference path="./analytics.d.ts" />

// 使用
analytics.track("page_view", { page: "/home" });
```

## DefinitelyTyped 与 @types

大多数流行库已有社区维护的类型定义，通过 `@types` 包安装。

### 安装类型定义

```bash
# 安装 lodash 的类型定义
npm install --save-dev @types/lodash

# 安装 node 的类型定义
npm install --save-dev @types/node

# 安装 express 的类型定义
npm install --save-dev @types/express
```

### 查找类型定义

```bash
# 搜索类型包
npm search @types/[库名]

# 或在 TypeSearch 网站搜索
# https://www.typescriptlang.org/dt/search
```

### 类型定义来源

类型定义包安装后会放在 `node_modules/@types/` 目录下，TypeScript 会自动识别。

## 三斜线指令

三斜线指令（Triple-Slash Directives）用于声明文件之间的依赖。

### /// <reference path="..." />

```ts
// types/base.d.ts
interface BaseConfig {
  apiUrl: string;
  timeout: number;
}

// types/app.d.ts
/// <reference path="./base.d.ts" />

interface AppConfig extends BaseConfig {
  environment: string;
}
```

### /// <reference types="..." />

```ts
// 引用 @types 包
/// <reference types="node" />

// 现在可以使用 NodeJS 类型
const env: NodeJS.ProcessEnv = process.env;
```

### /// <reference lib="..." />

```ts
// 引用内置 lib
/// <reference lib="dom" />

// 现在可以使用 DOM 类型
const element: HTMLElement = document.getElementById("app")!;
```

## 扩展现有类型

### 扩展模块

```ts
// 扩展 express 的 Request 接口
// types/express.d.ts
import "express";

declare module "express" {
  interface Request {
    user?: {
      id: string;
      role: string;
    };
  }
}

// 使用
import express from "express";

const app = express();

app.use((req, res, next) => {
  req.user = { id: "123", role: "admin" };
  next();
});

app.get("/", (req, res) => {
  console.log(req.user?.id); // ✅ 有类型提示
});
```

### 扩展全局对象

```ts
// types/global.d.ts
declare global {
  interface Window {
    __APP_VERSION__: string;
    __DEBUG__?: boolean;
  }
}

// 使用
console.log(window.__APP_VERSION__);
if (window.__DEBUG__) {
  console.log("Debug mode");
}

// 必须导出空对象才能使 declare global 生效
export {};
```

### 合并声明

```ts
// 原有定义
interface User {
  id: number;
  name: string;
}

// 扩展定义
interface User {
  email: string;
  role: "admin" | "user";
}

// 最终类型
const user: User = {
  id: 1,
  name: "Tom",
  email: "tom@example.com",
  role: "user"
};
```

## 声明文件组织

### 项目结构

```
project/
├── src/
│   ├── index.ts
│   └── utils.ts
├── types/
│   ├── global.d.ts      # 全局类型
│   ├── express.d.ts     # 扩展模块
│   └── my-lib.d.ts      # 第三方库类型
├── package.json
└── tsconfig.json
```

### tsconfig.json 配置

```json
{
  "compilerOptions": {
    "typeRoots": ["./types", "./node_modules/@types"],
    "types": ["node", "jest"]
  }
}
```

- `typeRoots`：指定类型定义的查找目录
- `types`：只包含指定的类型包（提高编译速度）

### 发布带类型的包

```json
// package.json
{
  "name": "my-awesome-lib",
  "main": "dist/index.js",
  "types": "dist/index.d.ts",
  "files": ["dist"]
}
```

## 常见问题

### 模块解析问题

```ts
// ❌ 找不到模块 "./utils" 的声明文件
import { greet } from "./utils";

// 解决方案1：添加 index.d.ts
// utils/index.d.ts
export function greet(name: string): string;

// 解决方案2：在 tsconfig.json 中配置
{
  "compilerOptions": {
    "allowJs": true,
    "declaration": true
  }
}
```

### 全局类型不生效

```ts
// global.d.ts
// ❌ 不生效
interface Window {
  myVar: string;
}

// ✅ 需要使用 declare global
declare global {
  interface Window {
    myVar: string;
  }
}
export {};
```

### 默认导出问题

```ts
// my-lib.d.ts
// ❌ 错误：ES 模块和 CommonJS 混用
export = MyLib; // CommonJS
export default MyLib; // ES 模块

// ✅ 正确：选择一种风格
// ES 模块风格
export default class MyLib {
  static init(): void;
}

// CommonJS 风格
declare class MyLib {
  static init(): void;
}
export = MyLib;
```

## 最佳实践

### 1. 优先使用 @types

```bash
# 先搜索是否有现成的类型定义
npm search @types/[库名]
```

### 2. 类型文件与源文件放在一起

```
src/
├── utils.ts
├── utils.d.ts    # 如果需要单独的声明文件
└── index.ts
```

### 3. 使用 d.ts 扩展而非覆盖

```ts
// ✅ 扩展现有类型
declare module "express" {
  interface Request {
    customField?: string;
  }
}

// ❌ 不要完全重新定义
// declare module "express" { ... }
```

### 4. 为未类型化的 npm 包创建声明

```ts
// types/untyped-lib.d.ts
declare module "untyped-lib" {
  // 先使用 any 快速原型
  export function doSomething(input: any): any;

  // 然后逐步精确类型
  export function doSomething(input: string): number;
}
```

### 5. 使用 unknown 而非 any

```ts
// ❌ 不安全
export function parse(json: string): any;

// ✅ 更安全
export function parse(json: string): unknown;

// 使用时进行类型检查
const result = parse(data);
if (typeof result === "object" && result !== null) {
  // ...
}
```

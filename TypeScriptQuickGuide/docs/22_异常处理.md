# 异常处理

TypeScript 的异常处理机制与 Java/C# 类似，但有一些重要差异。了解这些差异对于正确处理错误至关重要。

## try-catch-finally

### 基本语法

```ts
try {
  // 可能抛出异常的代码
  throw new Error("Something went wrong");
} catch (error) {
  // 处理异常
  console.error(error);
} finally {
  // 无论是否异常都会执行
  console.log("Cleanup");
}
```

### 与 Java/C# 的差异

**Java**：
```java
try {
  // ...
} catch (IOException e) {  // 可以指定异常类型
  // ...
} catch (SQLException e) { // 多个 catch 块
  // ...
}
```

**TypeScript**：
```ts
try {
  // ...
} catch (error) {  // 只能有一个 catch 块，不能指定类型
  // error 默认是 unknown（strict 模式下）
  if (error instanceof Error) {
    console.error(error.message);
  }
}
```

## Error 类型

### 内置 Error 类型

```ts
// 基本错误
const error = new Error("Something went wrong");

// 类型错误
const typeError = new TypeError("Expected a number");

// 范围错误
const rangeError = new RangeError("Value out of range");

// 引用错误
const refError = new ReferenceError("Variable not defined");

// 语法错误
const syntaxError = new SyntaxError("Invalid syntax");
```

### 自定义错误

```ts
// 方式1：继承 Error
class AppError extends Error {
  constructor(
    message: string,
    public code: string,
    public statusCode: number = 500
  ) {
    super(message);
    this.name = "AppError";

    // 修复原型链（TypeScript/ES6 需要）
    Object.setPrototypeOf(this, AppError.prototype);
  }
}

// 使用
throw new AppError("User not found", "USER_NOT_FOUND", 404);

// 方式2：带有更多信息的错误
class ValidationError extends Error {
  constructor(
    message: string,
    public errors: Record<string, string[]>
  ) {
    super(message);
    this.name = "ValidationError";
    Object.setPrototypeOf(this, ValidationError.prototype);
  }
}

// 使用
throw new ValidationError("Validation failed", {
  email: ["Invalid email format"],
  password: ["Password too short", "Password must contain numbers"]
});
```

### 错误类型判断

```ts
function handleError(error: unknown): never {
  if (error instanceof AppError) {
    console.error(`[${error.code}] ${error.message}`);
    throw error;
  }

  if (error instanceof ValidationError) {
    console.error("Validation errors:", error.errors);
    throw error;
  }

  if (error instanceof Error) {
    console.error(error.message);
    throw error;
  }

  // 处理非 Error 类型的异常
  console.error("Unknown error:", error);
  throw new Error("Unknown error occurred");
}
```

## throw 语句

### throw 任意值

与 Java/C# 不同，TypeScript/JavaScript 可以抛出任意类型的值：

```ts
// 可以抛出任意类型（但不推荐）
throw "Something went wrong";
throw 404;
throw { code: "ERROR", message: "Failed" };
throw null;

// 推荐做法：始终抛出 Error 或其子类
throw new Error("Something went wrong");
```

### 类型安全的抛出

```ts
// 定义错误类型
type Result<T, E = Error> =
  | { success: true; value: T }
  | { success: false; error: E };

function divide(a: number, b: number): Result<number> {
  if (b === 0) {
    return { success: false, error: new Error("Division by zero") };
  }
  return { success: true, value: a / b };
}

// 使用
const result = divide(10, 2);
if (result.success) {
  console.log(result.value);
} else {
  console.error(result.error);
}
```

## 异步异常处理

### Promise 中的异常

```ts
// Promise 链式调用
fetch("/api/users")
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.error("Error:", error))
  .finally(() => console.log("Done"));

// async/await
async function fetchUsers() {
  try {
    const response = await fetch("/api/users");
    if (!response.ok) {
      throw new Error(`HTTP error: ${response.status}`);
    }
    const data = await response.json();
    return data;
  } catch (error) {
    if (error instanceof Error) {
      console.error("Fetch error:", error.message);
    }
    throw error;
  } finally {
    console.log("Fetch completed");
  }
}
```

### 未捕获的 Promise 异常

```ts
// 处理未捕获的 Promise 异常
process.on("unhandledRejection", (reason, promise) => {
  console.error("Unhandled Rejection:", reason);
});

// 浏览器环境
window.addEventListener("unhandledrejection", (event) => {
  console.error("Unhandled Rejection:", event.reason);
  event.preventDefault();
});
```

### 正确处理异步异常

```ts
// ❌ 错误：忘记 await，异常不会被 catch 捕获
async function badExample() {
  try {
    fetchData();  // 没有 await
  } catch (error) {
    // 不会捕获到异常
  }
}

// ✅ 正确：使用 await
async function goodExample() {
  try {
    await fetchData();
  } catch (error) {
    // 可以捕获异常
  }
}

// ✅ 或使用 catch 链
async function alsoGood() {
  await fetchData().catch(error => {
    console.error(error);
  });
}
```

## Result 模式

作为异常处理的替代方案，使用 Result 类型可以更明确地处理错误：

```ts
// 定义 Result 类型
type Result<T, E = Error> =
  | { ok: true; value: T }
  | { ok: false; error: E };

// 工具函数
function Ok<T>(value: T): Result<T> {
  return { ok: true, value };
}

function Err<E>(error: E): Result<never, E> {
  return { ok: false, error };
}

// 使用示例
function parseUser(data: unknown): Result<User, string> {
  if (typeof data !== "object" || data === null) {
    return Err("Invalid data");
  }

  const obj = data as Record<string, unknown>;
  if (typeof obj.name !== "string") {
    return Err("Name is required");
  }
  if (typeof obj.email !== "string") {
    return Err("Email is required");
  }

  return Ok({ name: obj.name, email: obj.email });
}

// 处理结果
const result = parseUser(input);
if (result.ok) {
  console.log("User:", result.value);
} else {
  console.error("Error:", result.error);
}
```

## 与 Java/C# 的对比

| 特性 | Java | C# | TypeScript |
|------|------|-----|------------|
| 检查异常 | ✅ 必须 catch 或声明 | ❌ 无 | ❌ 无 |
| catch 类型指定 | ✅ 多个 catch 块 | ✅ when 子句 | ❌ 单一 catch |
| throw 类型 | Throwable 子类 | Exception 子类 | 任意类型 |
| finally | ✅ | ✅ | ✅ |
| try-with-resources | ✅ | using 语句 | ❌ 手动清理 |
| 异步异常 | CompletableFuture | async/await | Promise/async-await |

**Java**：
```java
// Java 检查异常
public void readFile() throws IOException {
  // 必须声明或捕获 IOException
}

try {
  readFile();
} catch (IOException e) {
  // 必须处理
}
```

**TypeScript**：
```ts
// TypeScript 无检查异常
function readFile(): void {
  // 可以抛出任何异常，编译器不强制处理
}

// 需要
try {
  readFile();
} catch (error) {
  // 手动处理
}
```

## 最佳实践

### 1. 始终抛出 Error 对象

```ts
// ❌ 不要抛出字符串
throw "Something went wrong";

// ✅ 抛出 Error 对象
throw new Error("Something went wrong");
```

### 2. 使用自定义错误类型

```ts
class DatabaseError extends Error {
  constructor(message: string, public query: string) {
    super(message);
    this.name = "DatabaseError";
  }
}

class NetworkError extends Error {
  constructor(message: string, public statusCode: number) {
    super(message);
    this.name = "NetworkError";
  }
}

// 处理时可以区分
try {
  // ...
} catch (error) {
  if (error instanceof DatabaseError) {
    console.error(`Database error: ${error.query}`);
  } else if (error instanceof NetworkError) {
    console.error(`Network error: ${error.statusCode}`);
  }
}
```

### 3. 使用 unknown 类型处理 catch

```ts
// ❌ error 默认是 any（非 strict 模式）
try {
  // ...
} catch (error) {
  console.log(error.message); // 不安全
}

// ✅ 使用类型守卫
try {
  // ...
} catch (error: unknown) {
  if (error instanceof Error) {
    console.error(error.message);
  } else if (typeof error === "string") {
    console.error(error);
  } else {
    console.error("Unknown error");
  }
}
```

### 4. 不要忽略异常

```ts
// ❌ 吞掉异常
try {
  doSomething();
} catch (e) {
  // 什么都不做
}

// ✅ 至少记录日志
try {
  doSomething();
} catch (error) {
  console.error(error);
  // 或重新抛出
  throw error;
}
```

### 5. finally 用于清理资源

```ts
let connection: Connection | null = null;

try {
  connection = await createConnection();
  await connection.query("SELECT * FROM users");
} catch (error) {
  console.error("Query failed:", error);
  throw error;
} finally {
  if (connection) {
    await connection.close();
  }
}
```

### 6. 使用 Result 模式处理预期错误

```ts
// 对于预期的错误，使用 Result 模式
function validateInput(input: string): Result<ValidInput, ValidationError> {
  // ...
}

// 对于意外的错误，使用异常
function connectDatabase(): Promise<Connection> {
  // ...
}
```
